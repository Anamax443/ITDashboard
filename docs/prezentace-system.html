IT Dashboard

Jednotný přehled o stavu počítačů, serverů, síťových zařízení a tiskáren — automatický sběr dat, včasné e-mailové výstrahy a manažerské souhrny na jednom místě.

Přehledová prezentace systému · anonymizovaná verze pro externí publikum
Proč systém vznikl

Od ručních kontrol k automatickému přehledu

🔒 Tato verze je anonymizovaná. Všechny názvy, IP adresy, jména zařízení a e-maily jsou nahrazeny generickými příklady (např. Pobočka A, 10.0.0.x, PC-001).

Správa rozsáhlejší IT infrastruktury (desítky až stovky počítačů, několik poboček, síťové prvky, tiskárny) se bez nástroje opírá o ruční kontroly a roztříštěné skripty. To znamená:

  • problémy se zjistí pozdě — až je nahlásí uživatel,
  • není jednotný přehled o tom, co v síti vlastně je,
  • opakovaná ruční práce (inventury, kontroly disků, služeb, tiskáren).

Cíl

Jeden dashboard, který sám sbírá data z více zdrojů, drží je v databázi, vizualizuje stav a proaktivně upozorní na problém dřív, než ho někdo nahlásí.

Sběr dat

Aplikační server periodicky čte data ze síťových prvků, z adresářové služby, pinguje zařízení a čte SNMP z tiskáren. Vše konfigurovatelné, bez tajemství v kódu.

Uložení

Strukturovaná SQL databáze s historií. Inventář je klíčovaný MAC adresou (trvalá identita) — IP je jen dočasná „adresa pobytu".

Prezentace

Webový dashboard s filtrací a manažerskými souhrny + automatické e-mailové reporty a výstrahy přes firemní SMTP relay.

Klíčová funkce

Síťový inventář: sloučení více zdrojů podle MAC

Žádný jediný zdroj nevidí všechno. Systém proto kombinuje několik pohledů a slučuje je podle MAC adresy do jednoho záznamu na zařízení:

ZdrojCo přidává
DHCP záznamydynamická i rezervovaná zařízení + jména
ARP tabulkastaticky adresovaná zařízení mimo DHCP
Aktivní skenjména (NETBIOS/DNS) i dosud neviděná zařízení

Duplicity jsou vyloučené z principu: jeden řádek na (lokalita, MAC), sloučení a doplňování dat, nikdy nový duplikát.

Robustní sběr

Data se čtou jednak živě přes API, jednak ze souborových snímků, které si síťový prvek sám pravidelně vytváří. Snímek funguje i jako signál aktuálnosti — když přestane „dýchat", systém pozná, že je něco špatně, a upozorní.

Identita = MAC

Zařízení si drží kategorii, jméno i historii IP adres, i když mu IP změní. „MAC = rodné číslo, IP = dočasná adresa."

Proaktivita

E-mailové výstrahy, ne hledání problémů

Místo ručního obcházení systém pošle strukturovaný e-mail, jakmile nastane definovaný stav. Každá výstraha má společný „rozumný" model:

  • Debounce — nepípne na chvilkový výpadek,
  • Throttling — připomene se v rozumném intervalu, ne zahltí,
  • Okno údržby — během plánovaných prací mlčí,
  • Per-lokalita / per-stroj — výjimky, aby „nekřičelo" to, co se teprve nasazuje.

Technologie, bezpečnost, provoz

Technologie

Node.js / TypeScriptREST API SQL databázeWebový frontend SNMPAdresářová služba SMTP

Provoz

  • běží jako služba na interním serveru,
  • automatické nasazení po schválené změně (CI: testy → build → migrace),
  • health-check endpoint pro externí monitoring.

Bezpečnost

  • Read-only servisní účty se síťovými prvky komunikují jen pro čtení,
  • přístup omezený na povolené adresy (allow-list),
  • hesla a tajemství šifrovaná, nikdy v kódu ani v repozitáři,
  • data zůstávají uvnitř firemní sítě.

Princip nejnižších oprávnění: i kdyby únik, servisní účet umí jen číst a jen z definovaných míst.
