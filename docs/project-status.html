Funkční rozsah

  • Eventlog napříč flotilou — Warning/Error/Critical z každého PC do centrální DB; filtrování, vyhledávání, časové grafy, žebříček „noisy" PC.
  • AD-synchronizovaný inventář — OS, poslední přihlášení, OU cesta, operátorem řízený monitor toggle (perzistuje napříč syncs).
  • Monitoring disků — periodický DCOM sken, konfigurovatelné prahy (% / GB), per-tier scope písmen disků.
  • Services collector + policy — Auto + neběžící služby, klasifikace drift vs. policy tabulka, ExitCode capture, GPO PS export.
  • Performance events — pomalý boot/shutdown/standby/resume z Diagnostics-Performance kanálu s pojmenovanými culprity.
  • Klasifikace dostupnosti — každý běh: online / offline / rpc_unavailable / access_denied.
  • Activity log — terminálový live view (filtr, pauza, copy) + perzistentní historie.
  • Per-PC Actions — vzdálené admin zkratky (MMC, services, eventvwr, RDP, admin shares) přes hardenované itd-* URL handlery.
  • Settings — frekvence, časové okno, zapnuté checky, prahy — aplikováno živě bez restartu služby.

Poslední změna — externalizace konfigurace (2026-06-11)

  • Browser klient přešel na relativní API URL (konec napevno zadané IP v buildu).
  • Installer: server přepisuje ITD_API_BASE podle hostu stažení.
  • LDAP login bez napevno zadané domény AXINETWORK.LOC.
  • .env.example přepsán jako přesný, kompletní config povrch (trusted-only SQL, GitHub Actions Variables).
  • Dokumentace sladěna: README, ARCHITECTURE, dashboard.html, SETUP-SERVER, HANDOFF.

Stack

  • Server — Node.js 20 + Fastify + TypeScript, Windows Service přes NSSM.
  • DB — MSSQL, Integrated Auth přes msnodesqlv8 (ne tedious — selhává „untrusted domain").
  • Klient — Electron + React + Vite + TypeScript.
  • Sběr dat — agentless přes RPC/MS-EVEN6 (Get-WinEvent) + DCOM (Get-CimInstance), TCP/135 pre-flight probe.
